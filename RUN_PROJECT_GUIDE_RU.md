# Полная инструкция: как запустить проект «КУБ» с нуля (очень подробно)

Ниже инструкция для Windows, максимально пошагово.

---

## 1. Что нужно установить заранее

1. **Node.js (LTS)**
2. **Git**
3. (Желательно) **VS Code**

---

## 2. Проверка, что всё установлено

Открой `cmd` (не PowerShell) и выполни:

```bat
node -v
npm -v
git --version
```

Если видишь версии (например `v20.x.x`, `10.x.x`, `git version ...`) — всё ок.

---

## 3. Скачивание проекта

Если проекта ещё нет на компьютере:

```bat
git clone https://github.com/SergeysYar/KybVision.git
cd KybVision
```

Если проект уже есть, просто перейди в папку:

```bat
cd C:\_Work\ProjectKub
```

---

## 4. Установка зависимостей

Внутри папки проекта выполни:

```bat
npm install
```

Дождись завершения (может занять 1–5 минут).

---

## 5. Запуск проекта в режиме разработки

Команда запуска:

```bat
npm run dev
```

Если вы запускаете из PowerShell и видите ошибку про `npm.ps1`, используйте:

```bat
cmd /c npm run dev
```

После запуска в консоли будет адрес, обычно:

```text
http://localhost:3000
```

Открой его в браузере.

---

## 6. Какие страницы проверить сразу

1. `/` — главная (landing)
2. `/opportunities` — возможности
3. `/ai` — AI-наставник
4. `/profile` — профиль
5. `/admin` — админка (если нужна для проверки mock-данных)

Пример:
`http://localhost:3000/opportunities`

---

## 7. Как остановить сервер

В окне, где запущен `npm run dev`, нажми:

```text
Ctrl + C
```

---

## 8. Если `npm` не запускается в PowerShell

У тебя может быть ошибка типа “execution of scripts is disabled”.  
Самый простой путь: запускать команды через **cmd**:

```bat
cmd /c npm run dev
```

Или просто всегда работать в обычном `Командная строка` (`cmd.exe`).

---

## 9. Проверка TypeScript (по желанию)

```bat
npx tsc --noEmit
```

Важно: в текущем проекте могут быть старые ошибки в других файлах, не связанные с последними правками.

---

## 10. Сборка production (по желанию)

```bat
npm run build
npm run start
```

Если увидишь конфликт `pages/...` и `app/...` — это архитектурный конфликт в проекте (не ошибка базового запуска dev-режима).

---

## 11. Мини-алгоритм “в 3 команды”

Если всё уже установлено и проект уже на диске:

```bat
cd C:\_Work\ProjectKub
npm install
npm run dev
```
