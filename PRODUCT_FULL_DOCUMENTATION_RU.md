# КУБ — полная документация по функционалу (MVP)

Документ описывает весь текущий функционал проекта: страницы, формы, логику, данные, AI-наставник, админку и ограничения.

## 1. Назначение платформы

`КУБ` — это MVP-платформа сопровождения проектных инициатив: от идеи до MVP.
Основные задачи:
- помочь пользователю определить трек развития;
- оформить карточку проекта;
- собрать команду;
- подобрать конкурсы/гранты/мероприятия;
- получить подсказки от AI-наставника;
- видеть прогресс в профиле и дашборде.

## 2. Карта разделов (роуты)

- `/` — landing (главная публичная страница)
- `/onboarding` — вводная страница
- `/test` — тест на определение трека
- `/dashboard` — центр управления проектом
- `/project` — карточка проекта
- `/team` — поиск/создание заявки в команду
- `/opportunities` — лента возможностей
- `/ai` — AI-наставник (сценарный FAQ-чат)
- `/profile` — профиль и прогресс пользователя
- `/admin` — demo-админка (локальное управление mock-данными)

## 2.1 Поведение навигации

- В боковом и нижнем меню активным подсвечивается только текущий раздел.
- При переходе в `/profile` активен только пункт `Профиль`.
- Остальные пункты меню остаются визуально читаемыми (не «гаснут» полностью), чтобы пользователь всегда мог быстро перейти в любой раздел.

## 3. Главная страница `/`

Что отображается:
- общий оффер платформы;
- CTA-кнопки «Начать» и «Открыть demo»;
- блок базового маршрута (`трек → проект → возможности`);
- блок «Локальные проектные инициативы».

Локальные инициативы (из `src/data/platformInfo.ts`):
- `CavkazVision`
- `KubanVision`

Роли в интерфейсе:
- Идеолог проекта «КУБ» — `Шаршак Алексей Александрович`;
- Проектное руководство инициатив `CavkazVision` и `KubanVision` — `Пантелеев`.

## 4. Карточка проекта `/project`

Назначение:
- создать и сохранить структурированное описание проекта в `localStorage`.

Форма включает поля:
- `Название проекта`
- `Кому нужен проект` (аудитория)
- `Проблема`
- `Решение`
- `Ценность`
- `Стадия проекта` (`idea`, `card`, `team`, `contest`, `mvp`)
- `Нужна ли команда` (`yes/no`)
- `Какие роли нужны` (при `needsTeam = true`)

Кнопки/действия:
- `Сохранить карточку`
- `Сбросить форму`
- `Удалить сохранённую карточку`

Валидация:
- обязательны `title`, `problem`, `solution`, `value`, `audience`;
- для `title` есть минимальная длина;
- показываются подсказки по коротким описаниям.

Дополнительно на странице:
- индикатор заполненности карточки (`%`);
- preview карточки;
- блок «Рекомендации по стадии»;
- быстрые переходы в `/opportunities`, `/team`, `/ai`.

## 5. Команда `/team`

Назначение:
- найти существующие заявки;
- создать или редактировать свою заявку.

Вкладки:
- `Все заявки`
- `Моя заявка`

Фильтр:
- фильтр по роли (`developer`, `designer`, `marketer`, `analyst`, `idea_author`, `mentor`).

Форма «Создать заявку»:
- `Ваша роль`
- `Кого ищете`
- `Описание заявки`
- `Контакты` (telegram/email)
- `Название проекта`
- `Навыки через запятую`

Мини-валидация:
- описание не короче порога;
- контакты обязательны.

Особенности:
- заявка хранится локально;
- можно редактировать, закрывать/переоткрывать, удалять;
- связь с участником через модальное окно;
- встроенного внутреннего чата нет (контакт напрямую).

## 6. Возможности `/opportunities`

Назначение:
- подбор событий и программ под трек и стадию.

Поддерживаются типы:
- `grant`
- `contest`
- `hackathon`
- `event`
- `club`
- `accelerator`

Фильтры:
- поиск по тексту;
- фильтр по типу;
- фильтр по треку;
- сортировка (ближайшие/поздние/по типу).

Секция рекомендаций:
- отдельный блок с релевантными возможностями для текущего трека.

Специальные инициативы в данных:
- `CavkazVision: цифровой туризм Кавказа`
- `KubanVision: туризм и маршруты Кубани`

В обеих описаниях явно указано:
- «Направление развивается под проектным руководством Пантелеева.»

## 7. AI-наставник `/ai`

Режим:
- сценарный FAQ-поиск по ключевым словам (`mockAiScenarios`);
- не LLM-генерация, а выбор подходящего сценария.

Интерфейс:
- чат;
- ручной ввод вопроса;
- быстрые кнопки вопросов;
- очистка истории чата.

История:
- хранится в `localStorage` (`kub_ai_chat_history`).

### 7.1 Быстрые вопросы на странице

- «Что такое MVP?»
- «Как оформить проект?»
- «Куда податься?»
- «Как найти команду?»
- «Что делать дальше?»

### 7.2 Поддерживаемые сценарии (основные темы)

- сбор команды и роли
- куда податься с идеей
- подготовка к хакатону
- подготовка питча
- грант vs конкурс
- следующий шаг
- fallback-ответ при нерелевантном вопросе

## 12. Demo-админка `/admin`

Назначение:
- локально управлять mock-данными и сценариями в MVP.

Основные вкладки:
- `Возможности`
- `Заявки команд`
- `Проекты`
- `AI-сценарии`

### 12.1 Форма возможностей

Поля:
- название;
- тип;
- описание;
- дата;
- URL;
- треки;
- теги;
- город;
- формат;
- сложность;
- публикация.

### 12.2 Форма AI-сценария

Поля:
- вопрос;
- ответ;
- категория;
- маршрут;
- ключевые слова (через запятую);
- label для быстрой кнопки (опционально).

## 13. Локальное хранилище (`localStorage`)

Ключи (из `STORAGE_KEYS`):
- `kub_user_track`
- `kub_completed_track_test`
- `kub_test_result`
- `kub_test_answers`
- `kub_project`
- `kub_my_team_request`
- `kub_ai_chat_history`
- `kub_hide_demo_banner`
- `kub_admin_opportunities`
- `kub_admin_team_requests`
- `kub_admin_projects`
- `kub_admin_ai_scenarios`

Используется для:
- сохранения состояния MVP без backend.

## 14. Структура данных (основные сущности)

- `Project`
- `TeamRequest`
- `Opportunity`
- `AiScenario`
- `User`
- `PlatformInfo` (инициативы и роли)

## 15. Текущие ограничения MVP

- данные в основном локальные (без реальной БД и API);
- AI-наставник сценарный, а не полноценный генеративный агент;
- нет внутреннего чата между участниками;
- production-сборка может требовать техдолг-исправлений в отдельных файлах, не связанных с базовым использованием MVP.

## 16. Быстрый пользовательский сценарий от нуля до результата

1. Открыть `/onboarding` и `/test`.
2. Перейти в `/project` и заполнить карточку.
3. Если нужна команда — создать заявку в `/team`.
4. Подобрать события в `/opportunities`.
5. Уточнять шаги в `/ai`.
6. Проверить прогресс и связанные инициативы в `/profile`.
